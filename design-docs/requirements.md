# Requirements — CIS Help Me Please

## Overview
ระบบแจ้งเหตุฉุกเฉินภายในมหาวิทยาลัย (CIS Help Me Please) เป็นเว็บแอปพลิเคชันที่ช่วยให้นักศึกษาและบุคลากรสามารถแจ้งเหตุฉุกเฉินได้อย่างรวดเร็ว พร้อมทั้งส่งข้อมูลไปยังเจ้าหน้าที่และติดตามสถานะการช่วยเหลือได้ทันท่วงที

## Goals & Non-Goals
- **Goals:**
  - สมัครสมาชิกและลงชื่อเข้าใช้งานสำหรับนักศึกษา/บุคลากร และเจ้าหน้าที่
  - แจ้งเหตุฉุกเฉินระบุประเภทเหตุการณ์ รายละเอียด สถานที่ พิกัด และแนบไฟล์ภาพได้
  - เจ้าหน้าที่สามารถดูรายการเหตุการณ์ อัปเดตสถานะ และบันทึกการช่วยเหลือได้
  - แจ้งเตือนเจ้าหน้าที่เมื่อมีเหตุใหม่และให้ผู้แจ้งติดตามสถานะได้
  - สรุปสถิติจำนวนเหตุแยกตามประเภทและเวลาตอบสนองการช่วยเหลือ
- **Non-goals:**
  - การประสานงานอัตโนมัติไปยังหน่วยงานภายนอกมหาวิทยาลัย (เช่น สถานีตำรวจ หรือสถานีดับเพลิง)
  - ระบบโทรเวชกรรม (Telemedicine) หรือการปฐมพยาบาลผ่านระบบวิดีโอคอล
  - การนำทางแบบ Real-time GPS นอกพื้นที่มหาวิทยาลัย

## Roles & Actors (บทบาทผู้ใช้งาน)
ระบบแบ่งผู้ใช้งานออกเป็น 3 บทบาทหลัก ดังนี้:
1. **ผู้แจ้งเหตุ (Reporter - นักศึกษา/บุคลากร):** ผู้พบเห็นเหตุการณ์ฉุกเฉิน สามารถสมัครสมาชิก เข้าสู่ระบบ แจ้งเหตุฉุกเฉินพร้อมแนบรายละเอียด/พิกัด/ภาพถ่าย และติดตามสถานะช่วยเหลือของตนเอง
2. **เจ้าหน้าที่ช่วยเหลือ (Staff - เจ้าหน้าที่/ผู้ดูแลระดับปฏิบัติการ):** เจ้าหน้าที่ที่รับผิดชอบการเข้าช่วยเหลือในมหาวิทยาลัย สามารถเข้าสู่ระบบ ดูรายการเหตุฉุกเฉินทั้งหมด รับเรื่องแจ้งเหตุเพื่อดำเนินการ บันทึกการดำเนินงาน และปิดงาน (ปรับสถานะเป็นเสร็จสิ้น)
3. **ผู้บริหาร / ผู้ดูแลระบบ (Admin - ผู้บริหาร/ผู้จัดการความปลอดภัย):** ผู้ดูแลระบบภาพรวม สามารถจัดการข้อมูลและสิทธิ์ของผู้ใช้/เจ้าหน้าที่ และเข้าดูรายงานสถิติต่าง ๆ เช่น สรุปจำนวนเหตุแยกตามประเภท และสถิติเวลาที่ใช้ในการตอบสนอง

## User Stories
- ในฐานะนักศึกษา/บุคลากร ฉันต้องการสมัครสมาชิกและแจ้งเหตุฉุกเฉินพร้อมระบุพิกัดและรูปถ่าย เพื่อให้เจ้าหน้าที่เข้ามาช่วยเหลือได้อย่างถูกต้องและรวดเร็ว
- ในฐานะผู้แจ้งเหตุ ฉันต้องการติดตามสถานะการดำเนินการช่วยเหลือ เพื่อความอุ่นใจและทราบความคืบหน้า
- ในฐานะเจ้าหน้าที่ช่วยเหลือ ฉันต้องการรับแจ้งเตือนเหตุฉุกเฉินใหม่ทันที เพื่อเข้าช่วยเหลือได้ทันท่วงที
- ในฐานะเจ้าหน้าที่ช่วยเหลือ ฉันต้องการอัปเดตสถานะและบันทึกข้อมูลการช่วยเหลือ เพื่อเป็นหลักฐานการทำงาน
- ในฐานะผู้บริหาร/เจ้าหน้าที่ ฉันต้องการดูรายงานสถิติจำนวนเหตุและเวลาตอบสนอง เพื่อวางแผนและปรับปรุงมาตรการความปลอดภัย

## Functional Requirements
| ID | Requirement | Priority |
|----|-------------|----------|
| FR-1 | ผู้ใช้งานสามารถลงทะเบียนด้วย ชื่อ, เบอร์โทร, รหัสนักศึกษา, และสถานที่พัก | Must |
| FR-2 | ผู้ใช้งานสามารถเข้าสู่ระบบและออกจากระบบได้ | Must |
| FR-3 | ผู้ใช้งานสามารถแจ้งเหตุฉุกเฉินโดยระบุประเภท รายละเอียด สถานที่ พิกัด และแนบรูปภาพได้ | Must |
| FR-4 | เจ้าหน้าที่สามารถเข้าดูรายการแจ้งเหตุทั้งหมดและกรองตามสถานะได้ | Must |
| FR-5 | เจ้าหน้าที่สามารถอัปเดตสถานะเหตุการณ์ (รอการตอบสนอง, กำลังดำเนินการ, เสร็จสิ้น) และบันทึกการช่วยเหลือได้ | Must |
| FR-6 | ระบบส่งการแจ้งเตือนไปยังเจ้าหน้าที่เมื่อมีเหตุฉุกเฉินใหม่ | Should |
| FR-7 | ผู้แจ้งเหตุสามารถเข้ามาดูและติดตามสถานะเหตุการณ์ที่ตนเองแจ้งได้ | Must |
| FR-8 | ระบบสามารถแสดงสถิติสรุปจำนวนเหตุการณ์ตามประเภทและเวลาตอบสนองเฉลี่ยได้ | Should |

## Non-Functional Requirements
| ID | Category | Requirement |
|----|----------|-------------|
| NFR-1 | Security | ข้อมูลรหัสผ่านของผู้ใช้งานต้องถูกเข้ารหัสอย่างปลอดภัย และจำกัดการเข้าถึงข้อมูลเจ้าหน้าที่เฉพาะผู้มีสิทธิ์ |
| NFR-2 | Usability | หน้าจอแอปพลิเคชันต้องรองรับการแสดงผลบนอุปกรณ์พกพา (Responsive Design) เพื่อความสะดวกในการใช้งานขณะเกิดเหตุ |
| NFR-3 | Performance | ระบบต้องสามารถส่งข้อมูลแจ้งเหตุและแจ้งเตือนไปยังเจ้าหน้าที่ได้ภายในเวลาไม่เกิน 3 วินาที |

## Constraints & Assumptions
- พัฒนาเว็บแอปพลิเคชันโดยใช้ Next.js 16 (App Router) และ React 19 ตามโครงสร้างปัจจุบัน
- ใช้ Tailwind CSS v4 สำหรับจัดแต่งดีไซน์ตามมาตรฐานระบบ
- สมมติให้ผู้ใช้เปิดใช้งานสิทธิ์การเข้าถึงพิกัดระบุตำแหน่ง (Geolocation) ผ่านเบราว์เซอร์

## Open Questions
- วิธีการจัดการสิทธิ์ของเจ้าหน้าที่ (เช่น กำหนดจากรหัสพนักงาน หรือมีระบบอนุมัติบัญชีเจ้าหน้าที่แยกต่างหาก)
- ช่องทางการแจ้งเตือนเจ้าหน้าที่นอกเหนือจากหน้าจอแอปพลิเคชัน (เช่น Line Notify, Email หรือ Push Notification บนมือถือ)
